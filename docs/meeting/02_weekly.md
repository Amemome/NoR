## 1. 서론

NoR 프로젝트는 논문, 발표, 학술 보고서 등에서 사용할 수 있는 한글 기반 초경량 시각화 스크립트 언어를 개발하는 것을 목표로 하고 있다. 해당 언어는 복잡한 시각화 도구(matplotlib, R 등)를 대신하여 간단하고 직관적인 명령어만으로 고품질 그래프를 생성할 수 있는 DSL(Domain-Specific Language)로 설계 중이다.

이번 주차에는 본격적인 개발에 들어가기에 앞서 공통 문법 정의 문서 작성, 역할 분담 확정, GitHub 협업 구조 설정, 초기 실행 환경 세팅 등의 기반 작업을 정하였고 그 후에 본격적인 작업이 시작되었다. 이는 향후 각 파트에서 혼선 없이 구현을 진행하기 위한 기준점이 되는 주차이며, 특히 NoR의 스크립트 문법을 구체화하고 Lark 파서 구현의 기반이 되는 문서 초안을 확정하는 것이 가장 중요한 성과였다.

또한 GUI, Graph 실행기 등 기술 파트별 책임을 분명히 하고, 각자가 개발을 시작할 수 있도록 사전 작업들을 체계화하였다.

이 보고서에서는 2주차에 이루어진 회의 논의 내용과 주요 결정사항, 그리고 각 파트별 진행 계획과 구현 준비 현황을 정리하고자 한다.

## 2. 2주차 회의 개요 및 목표

이번 주차의 작업을 위한 회의는 NoR 문법 정의 문서를 산출하기 위한 기반 설계 회의로, 역할 분담, GitHub 초기화, 개발 환경 통일, 커밋 및 브랜치 규칙 정립 등 향후 본격적인 구현을 위한 공통 기준을 확립하는 데에 중점을 두었다.

회의는 2025년 5월 8일 오후 5시 30분, 단국대 도서관 스터디룸에서 대면으로 진행되었으며, 모든 팀원이 참석하여 주요 사안에 대해 논의하고 결정하였다.

회의는 다음과 같은 절차로 진행되었다:

1. GitHub 레포지토리 구조 공유 및 디렉토리 구조 합의
2. 문법 정의 문서의 구성 항목 제안 및 확정

6. 파트별로 이번 주 안에 해야 할 작업 정리 및 공유

회의 후 각 팀원은 자신의 역할에 맞는 구현 작업을 개별적으로 진행하고 진행 상황은 온라인으로 공유하였으며, 5월 11일에 비대면 줌 회의를 진행하였다. 그 후에는 각각 이를 바탕으로 파서 작동 실험, 그래프 출력 함수 정의, GUI 입력창 연결 데모 등을 실현을 시작하였다.

## 4. 각 파트별 진행 상황

각 파트별 담당자가 본격적인 구현에 앞서 기초 작업 및 준비 단계를 진행하였다. 이 단계에서 각자의 개발 방향성과 필요 조건을 정리하고, 실제 코딩 이전에 공통 문법과 명령 체계에 대한 이해를 맞추는 데 집중하였다.

### 4.1 Language 파트 (담당: 류지성)

Language 파트는 NoR의 핵심인 스크립트 문법 정의 및 파서 구현을 담당하고 있다. 2주차에는 문법 정의 문서 구조를 검토하고, Lark 기반 문법 파일 생성을 준비하였다.

#### 주요 준비사항
- EBNF 스타일의 문법 정의 기준을 docs/NoR_문법정의_초안.md 문서에 정리 중이다.
- 명령어 호출(그리기(x, y)), 변수 선언(x = [1, 2, 3]) 등 핵심 구문 패턴을 정리하였다.
- 한글 키워드의 다중 표현 대응 방식(예: 그리기, draw, 그래프그리기)을 AST 변환 기준으로 설계 중이다.

#### 구현 계획
- 3주차에는 Lark 문법 파일(nor.lark)을 기반으로 간단한 스크립트를 파싱하고, 추상 구문 트리(AST)를 출력하는 테스트 코드를 작성할 예정이다.
- AST 구조는 각 노드를 명령어 단위로 변환한 딕셔너리 형태로 리턴하여, 실행 파트와 연결 가능하게 설계할 계획이다.

### 4.2 Graph 파트 (담당: 이운길)

Graph 파트는 사용자 명령어를 실제 그래프 출력으로 연결하는 실행기를 구현하는 역할을 맡고 있다. matplotlib 기반으로 선 그래프, 막대그래프, 산점도 등 주요 그래프 유형을 처리할 수 있는 함수 구조를 구상하고 있다.

#### 주요 준비사항
- 각 그래프 유형에 대응되는 함수 틀(draw_line, draw_bar, draw_scatter)을 설계하였다.
- 색상, 폰트, 저장 경로 등을 기본 설정하는 style_config.py 모듈을 작성할 예정이다.
- 그래프 명령어는 내부적으로 영어 키워드로 통일된 형태(draw, set_title, save)로 받아 처리할 수 있도록 설계 중이다.

#### 구현 계획
- 3주차에는 예시 데이터를 바탕으로 그래프 3종류 출력 함수 구현을 완료하고, 이후 AST에서 전달받은 명령어 dict를 기반으로 그래프를 생성할 수 있도록 연동 구조를 테스트할 예정이다.
- 색상 설정, 저장 파일명 자동 생성 기능도 함께 포함할 계획이다.

### 4.3 GUI 및 문서화 파트 (담당: 최현경)

GUI 및 문서화 파트는 사용자 입력 인터페이스와 문서 관리 체계를 담당하고 있다. NoR 언어가 실제로 동작하는 모습을 확인할 수 있도록, PySimpleGUI 기반의 입력-출력 창 틀을 설계하고 있다.

#### 주요 준비사항
- GUI 구조를 /gui/ 디렉토리에 구성 중이며, main.py에서 실행 가능한 구조로 설계하고 있다.
- 스크립트 입력창, 실행 버튼, 그래프 출력창을 포함하는 레이아웃을 구상하였다.
- 테스트용 예시 스크립트 및 에러 메시지 구조도 함께 정리 중이다.

#### 구현 계획
- 3주차에는 GUI 상에서 스크립트를 입력받아 실행하는 임시 데모 구조를 완성하고자 한다.
- 아직 파서와 실행기가 완성되지 않았기 때문에, GUI 내부에서는 print("실행 중") 등의 임시 출력을 활용해 테스트할 예정이다.
- 에러 발생 시 사용자에게 메시지를 표시할 수 있는 예외 처리 창 구성도 고려 중이다.

## 5. 초기 디렉토리 및 예시 구조

프로젝트의 체계적인 개발을 위해 GitHub 레포지토리의 디렉토리 구조를 사전에 정리하였다. 각 파트별로 독립적인 작업 공간을 제공하며, 향후 병합 시 폴더 단위 책임 구조를 유지할 수 있도록 설계되었다.

### 디렉토리 구조 개요

```
/nor/
├── /gui/      # 사용자 인터페이스 (PySimpleGUI)
├── /lang/     # 파서 및 인터프리터 (Lark 문법, AST 처리)
├── /graph/    # 그래프 실행기 (matplotlib, seaborn)
├── /examples/ # 예시 스크립트 모음
├── /docs/     # 문법 정의, 계획서, 회의록 등 문서 모음
└── README.md  # 프로젝트 개요, 실행 방법, 문법 예시 포함
```

### 각 디렉토리 설명

| 디렉토리 | 설명 |
|----------|------|
| gui/ | PySimpleGUI 기반 GUI 구성 파일을 저장하며, main.py는 기본 실행 진입점으로 사용함 |
| lang/ | Lark 기반의 .lark 문법 파일, 파서 로직, AST 출력 코드 등을 관리함 |
| graph/ | matplotlib 및 seaborn 기반 그래프 그리기 함수, 스타일 설정, 저장 기능 등을 포함함 |
| examples/ | NoR 명령어 예시 스크립트 파일(.nor 또는 .txt), 테스트 데이터 샘플을 저장함 |
| docs/ | 문법 정의 문서, 회의록, 보고서, 개발 계획서 등을 저장하는 문서 전용 폴더임 |
| README.md | 프로젝트 설명, 설치 방법, 주요 명령어 예시 등을 포함하는 진입 문서임 |

### 관리 기준
- 모든 파트는 해당 디렉토리 내에서 브랜치별로 작업하고 병합 시 PR 기반으로 통합한다.
- 공통 문서(docs/NoR_문법정의)는 각 파트에서 참조하며, 변경 시 반드시 리뷰를 거쳐 반영한다.
- 테스트용 스크립트(examples/)는 기능 구현 시 우선적으로 활용되며, 향후 발표 및 문서화 단계에서 예시로 사용될 수 있도록 꾸준히 정리한다.

이 구조는 이후 개발 확장 및 발표 자료 제작 시에도 유용하게 사용될 예정이며, 추가되는 기능에 따라 하위 디렉토리를 유동적으로 분할할 수 있도록 유연하게 설계되어 있다.

## 6. 향후 계획 및 다음 회의 준비 사항

이번 주에는 문서와 구조 중심의 준비가 주를 이루었지만, 다음 주에는 실질적인 코드 작성 및 결과 확인에 중점을 두게 된다.

### 파트별 구현 목표

| 파트 | 구체적 목표 |
|------|------------|
| Language | - .lark 문법 파일을 작성하여 실제 스크립트 파싱 테스트를 수행함<br>- 추상 구문 트리(AST) 출력 구조를 확인하고, 그래프 파트와 연결 가능한 딕셔너리 형태로 변환 방안을 실험함 |
| Graph | - draw_line, draw_bar, draw_scatter 함수를 정의하여 입력 데이터만으로 기본 그래프를 생성하고 저장하는 실행기 로직을 구현함<br>- matplotlib의 스타일 설정 모듈(style_config.py)을 작성함 |
| GUI | - gui/main.py를 작성하여, 입력창·실행 버튼·출력 이미지 창이 포함된 기본 틀을 구성함<br>- 아직 파서 및 그래프 연결이 완성되지 않았기 때문에, 임시 출력(print("실행")) 등으로 흐름 테스트를 진행함 |
| 공통 문서화 | - docs/NoR_문법정의_초안 문서를 업데이트하고, 키워드 목록/자료형/예시 문법 등을 보완함 |

### 다음 회의 준비 사항

다음 회의는 오프라인으로 진행할 예정이다. 회의에서 다룰 핵심 사항은 다음과 같다:

| 항목 | 내용 |
|------|------|
| 1. AST 출력 확인 | Language 파트에서 .lark 문법 기반 파싱 결과를 시연하며 AST 구조를 점검함 |
| 2. 그래프 시각화 테스트 | Graph 파트에서 예시 데이터를 기반으로 선/막대/산점도 그래프를 성공적으로 출력했는지 확인함 |
| 3. GUI 연결 데모 | GUI 파트에서 스크립트 입력 → 버튼 클릭 → 실행 출력의 흐름이 정상적으로 구현되었는지 점검함 |
| 4. 문법 정의 보완 | 문법 정의 문서에서 누락된 항목이나 잘못된 규칙이 있는지 점검하고 보완함 |
| 5. 커밋/브랜치 리뷰 | 각 파트별 PR 히스토리를 점검하고, 커밋 메시지 및 리뷰 절차가 적절히 이루어졌는지 확인함 |

이번 회의와 작업을 통해 팀은 공통된 언어 정의 기준과 협업 체계를 정립하였으며, 이제부터는 구체적인 구현 작업과 피드백 과정을 반복하면서 NoR 프로젝트를 점차 실질적인 도구로 완성해나갈 예정이다.
