# NoR 문법 정의 초안

본 문서는 NoR 스크립트 언어의 핵심 문법 및 설계 기준을 문서화한 것이다.  
Language, Graph, GUI 파트 모두 본 문서를 기준으로 구현을 진행해야 한다.

---

## 1. 설계 철학 (Design Philosophy)

NoR은 한글 기반의 초경량 스크립트 언어로,  
복잡한 시각화 라이브러리 없이도 누구나 간단한 명령어만으로 고품질 그래프를 생성할 수 있도록 설계된다.

- 학술 보고서, 논문, 발표 자료용 시각화를 타겟으로 함
- 문법은 최대한 직관적이며, 한글로 작성 가능해야 함
- 코딩에 익숙하지 않은 사용자가 쉽게 적응가능해야 함
- 범용 언어가 아닌 그래프 생성에 최적화된 DSL(Domain Specific Language)로 정의됨

---

## 2. 키워드 정의

| 명령어 | 매핑 함수 | 파라미터 | 설명 |
|--------|-----------|-----------|------|
| 그리기 | draw | x, y 리스트 | 선형/산점도/막대그래프 그리기 |
| 제목설정 | set_title | 문자열 | 그래프 제목 지정 |
| 축이름설정 | set_labels | 문자열 2개 | x축, y축 이름 설정 |
| 저장하기 | save | 문자열 | 이미지 저장 |
| 그래프종류설정 | set_type | 문자열 | 선/막대/산점도 설정 |
| 색상설정 | set_color | 문자열 | 그래프 색상 설정 |

### 키워드 동의어

- `그리기`, `그래프`, `draw` → 같은 키워드
- `제목`, `제목설정` → set_title 이라는 rule 로 인식된다.

## 3. 변수 규칙

| 항목 | 규칙 |
|------|------|
| 변수명 | 한글, 영문, 숫자 조합 가능 (숫자 시작 금지) |
| 구분 | snake_case 권장 |
| 공백 | 변수명에 공백 불가 |
| 선언 형식 | 변수 = [1, 2, 3] 또는 변수 = "문자열" 또는 변수는 "문자열" |

---

## 4. 자료형

- 숫자(NUMBER): 부동소수점 배정도 형식의 소수 (예: 1, 3.14)
- 문자열(STRING): 큰 따옴표 또는 작은 따옴표로 감싼다.`"텍스트"` `'텍스투'`
- 불리언(BOOLEAN): 참, 거짓의 값을 가진다. 
- 배열/벡터: 대괄호 `[]`로 표현된 배열. 각 요소는 ,(쉼표)로 분리된다. (배열 안에 배열을 가질 수 있다.)

---

## 5. 구문 규칙
NoR 스크립트에서 사용되는 구문들의 형식과 예시를 설명합니다. 모든 구문은 하나 이상의 문장`statement`로 구성되며, 각 문장의 구분은 한 줄로 이루어집니다.

### 5.1 그래프 생성
새로운 그래프 작업을 시작하거나 현재 작업할 그래프를 지정합니다. 이 선언 이후의 속성 설정 및 그리기 명령은 이 그래프에 적용됩니다.

설명
```
그래프생성 "그래프 이름 또는 설명"
```

- 그래프생성: 새로운 그래프 컨텍스트를 시작하는 키워드입니다. (동의어: 없음 - 필요시 추가)
- "그래프 이름 또는 설명" : 생성되는 그래프를 식별하거나 부가적인 설명을 위한 문자열입니다. 이 문자열은 GUI의 제목 표시 등에 활용될 수 있습니다.

예시
```
그래프생성 "월별 온도 변화 그래프"
그래프생성 "매출 분석 리포트"
```

### 5.2 그래프 속성 정의
```
<속성명> <연결자> <속성값>

예시: x축은 "온도"
```
- 속성명: 그래프의 속성을 지정하는 규칙
- 연결자: 속성명과 값을 명시적으로 연결한다. 자연어스러운 느낌을 추가한다.
- 속성값: 속성명에 종속적인 값들. 보통 STRING이거나 숫자가 된다.

### 5.3 그래프 그리기 (실행 트리거)
현재 설정된 그래프 컨텍스트의 속성과 데이터를 바탕으로 실제 그래프를 생성(렌더링)합니다.
```
그리기
```
사용 가능한 키워드
- 그리기
- draw
- 그려줘
- 그려

### 5.4 연결자
모든 `그래프 속성 정의` 구문에서 `<속성명>`과 `<속성값>` 사이에는 다음과 같은 연결자가 사용될 수 있습니다.

*   `은`
*   `는`
*   `=`
*   `<-`

이 연결자들은 문법적으로 동일하게 취급되며, 사용자에게 자연스러운 표현을 선택할 수 있는 유연성을 제공합니다.

## 6. 주석
- 한 줄 주석: `//` 부터 해당 줄 끝까지
- 여러 줄 주석: `/*` 와 `*/` 사이의 모든 내용
- 주석은 렉싱 과정에서 무시됩니다.


## 7. 구체적인 속성명

`그래프 속성 정의` 구문(` <속성명> <연결자> <속성값> `)에서 사용될 수 있는 구체적인 `<속성명>` 키워드들과 그에 해당하는 `<속성값>`의 구체적인 명세입니다.

### 7.1 제목 (Title)

그래프의 전체 제목을 설정합니다.

**사용 가능한 키워드 (`<속성명>`):**

- `제목`
- `타이틀`
- `title`

**`<속성값>`:**

문자열 (STRING)

**예시:**

```
제목 은 "월별 평균 강수량"
타이틀 = "Monthly Average Precipitation"
title <- "2024년 1분기 실적"
```

### 7.2 그래프 종류 (Type)

생성할 그래프의 종류를 지정합니다.

**사용 가능한 키워드 (`<속성명>`):**

- `종류`
- `타입`
- `type`
- `유형`
- `그래프종류`

**`<속성값>`:**

- 미리 정의된 그래프 종류 문자열
    -   `"선"`
    -   `"막대"`
    -   `"산점도"`
    -   (동의어 또는 영어 추가 가능: `"라인"`, `"바"`, `"스캐터"`)

**예시:**

```
종류 는 "선"
타입 = "막대"
type <- "산점도"
```

### 7.3 X축 이름 (X-axis Label)

그래프의 X축에 표시될 이름을 설정합니다.

**사용 가능한 키워드 (`<속성명>`):**

- `x축`
- `엑스축`
- `xlabel`

**`<속성값>`:**

- 문자열 (STRING)

**예시:**

```
x축 은 "시간 (월)"
엑스축 = "측정 순서"
xlabel <- "Sample Index"
```

### 7.4 Y축 이름 (Y-axis Label)

그래프의 Y축에 표시될 이름을 설정합니다.

**사용 가능한 키워드 (`<속성명>`):**

- `y축`
- `와이축`
- `ylabel`

**`<속성값>`:**

- 문자열 (STRING)

**예시:**

```
y축 은 "온도 (°C)"
와이축 = "값 (단위)"
ylabel <- "Value (Unit)"
```

---

**(향후 확장 고려) 축 이름 동시 설정**

*   만약 `축이름 은 "X축 설명", "Y축 설명"`과 같이 X축과 Y축 이름을 한 번에 설정하는 기능을 지원한다면, 해당 키워드 (`PROP_LABELS`에 해당)와 두 개의 문자열을 받는 값 형식을 여기에 명시할 수 있습니다.
    *   **예상 키워드:** `축이름`, `레이블`, `labels`
    *   **예상 값 형식:** `"X축 문자열", "Y축 문자열"`

---

**(향후 확장 고려) 기타 주요 속성들**

아래는 향후 NoR 언어에 추가될 수 있는 주요 그래프 속성들의 예시입니다. 필요에 따라 각 속성에 대한 키워드와 값 형식을 위와 같은 방식으로 정의할 수 있습니다.

*   **색상 설정 (Color)**
    *   예상 키워드: `색`, `색상`, `color`
    *   예상 값: `"파랑"`, `"#FF0000"`, `"red"` (문자열 또는 특정 색상 코드 형식)
*   **글꼴 설정 (Font)**
    *   예상 키워드: `글꼴`, `폰트`, `font`
    *   예상 값: `"나눔고딕"`, `"Arial"` (문자열 - 폰트 이름), 추가적으로 폰트 크기 (숫자)
*   **그래프 크기 설정 (Size/Figure Size)**
    *   예상 키워드: `크기`, `그래프크기`, `size`, `figsize`
    *   예상 값: `800, 600` (숫자 2개 - 가로, 세로 픽셀 등)
*   **범례 설정 (Legend)**
    *   예상 키워드: `범례`, `legend`
    *   예상 값: `참`/`거짓` (표시 여부), 위치 문자열 (예: `"오른쪽 위"`)
